# 正职工考勤多维报表

本项目支持两种模式：

- 本地脚本模式：读取本地 Excel 文件并生成报表
- 公网网站模式：上传数据后在线生成并查看报表（Streamlit）

## 输入文件（最新版本）

- `0515排班/每日打卡工时推送模版.xlsx`：主明细数据主体
- `0515排班/美区签字报表 (13).xlsx`：用于累计总工时合并

## 已实现规则（按 2026-05-18 新版）

- 班次名称为空：标记 `未排班`
- 班次上班时间与首打卡时间差值在 `±1小时` 内：标记 `排班正确`，超出标记 `排班不正确`
- 节假日/休息日：无首末卡标记 `正确`；有打卡按正常班次规则判断
- 有打卡但首末卡不完整：标记 `不正确`
- 工作日备注包含 `休假/公出/出差`：标记 `正确`
- 备注包含 `居家办公`：按“首末卡时长+居家流程时长是否覆盖班次时长”判断
- 日超8h人数：`每日总工时 + 居家办公合计（审批中） - 8 > 0`
- 签字报表累计总工时字段并入主表（若缺失则回退主表累计总工时）
- 本周加班工时：`累计总工时 - 40`
- 缺卡次数：休息日和提交流程人员剔除后，按 `max(4-班次内打卡次数,0)` 统计

## 报表统计维度

- 排班正确率 = 排班正确人数 / 总人数
- 日超8h人数
- 排班率 = 有排班人数 / 总人数
- 缺卡率 = 缺卡次数之和 / (应打卡人数 × 4)
- 中后台人员按三级部门归类，其他人员按四级部门归类
- 颜色标识：
  - 排班率：100%绿、90%-99%黄、90%以下红
  - 排班正确率：90%以上绿、50%-90%黄、50%以下红
  - 缺卡率：90%以上红、50%-90%黄、50%以下绿

## 运行方式

```bash
python3 generate_report.py
```

## 公网交互网站（上传 -> 生成 -> 查看）

### 本地启动

```bash
pip install -r requirements.txt
streamlit run app.py
```

启动后浏览器打开：

- `http://localhost:8501`

### 网站交互能力

- 上传 2 个必传 Excel：
  - 主明细
  - 签字报表
- 可选上传规则文档（docx，留档用）
- 点击“生成报表”后可：
  - 在线查看完整 HTML 报表
  - 下载 `attendance_report.html`
  - 下载 `attendance_report_data.json`

### 输入校验与错误提示

- 必传文件缺失会阻止生成
- 文件超过 50MB 会提示超限
- 若缺少必需工作表（sheet），会提示具体缺失项

## 口径说明

- 制作日期：默认推断为主明细中最大考勤日期 `+1` 天
- 缺卡率：`缺卡次数/标准打卡数`（分母为 0 返回 `N/A`）
- 排班率：`是否排班=是` 的记录数 / 总记录数
- 排班正确率：`是否排班正确=正确` 的记录数 / 总记录数
- 每日总工时：优先使用源表已有总工时列，再叠加 `居家办公合计（审批中）`

## 本次任务复盘与后续改进

- 当前文件中未提供花名册，导致“新入职剔除”只能基于离职流程中的入职日期字段实现，建议补充花名册源文件后改为双源校验。
- “国内员工”定义在源数据中缺少显式标识，目前 `GL00` 规则按工号前缀执行，建议新增国家/用工地字段避免误剔除。
- 目前手工流程时段（半天/全天）规则缺少结构化字段，暂未自动还原，建议后续补充流程起止时间字段后纳入自动计算。
